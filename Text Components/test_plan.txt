1. SCOPE

In Scope

This project focuses on testing two application layers:

API Testing
Application: JSONPlaceholder API

Endpoints Tested:
- GET /posts
- POST /posts
- PUT /posts/{id}
- DELETE /posts/{id}
- Error handling scenarios (404 responses)

UI Testing
Application: OrangeHRM

Features Tested:
- Login functionality
- Dashboard navigation
- Employee management workflows
- Form interactions
- Validation messages
- User interface element verification

Out of Scope

- Database validation
- Security and penetration testing
- Source code review
- Production environment testing
- Mobile application testing
- Performance testing of OrangeHRM

2. TEST OBJECTIVES

The primary objective of this testing effort is to verify that both the API and UI layers function correctly and meet expected requirements.

Specific objectives include:

- Validate CRUD operations within the JSONPlaceholder API.
- Verify accurate API responses, status codes, and data structures.
- Ensure OrangeHRM user workflows operate correctly.
- Confirm UI elements behave as expected.
- Identify defects affecting functionality and usability.
- Demonstrate automation skills using Selenium and Java.
- Validate application quality through manual and automated testing.

3. TESTING LEVELS

Unit Testing

- Existing application unit tests are outside the testing scope.
- API endpoint validation will simulate component-level verification.

Integration Testing

- Verification that API requests and responses function correctly.
- Validation of interactions between application components.

System Testing

- End-to-end testing of OrangeHRM user workflows.
- Full API operation validation.

User Acceptance Testing (UAT)

- Validation that core business functionality behaves as expected from an end-user perspective.

4. TEST TYPES

Functional Testing

- Verify application features behave according to requirements.

API Testing

- Validate HTTP methods, responses, status codes, and payloads.

UI Automation Testing

- Automate critical user workflows using Selenium WebDriver.

Regression Testing

- Ensure existing functionality remains stable after changes.

Negative Testing

- Verify proper handling of invalid inputs and unexpected conditions.

Accessibility Testing

- Keyboard navigation testing.
- Basic WCAG compliance observations.
- Colour contrast review.

Cross-Browser Testing

- Verify functionality across different browsers where applicable.

7. RISKS

Risk 1: Third-Party Service Availability

Description:
JSONPlaceholder is a public API and may experience outages or response delays.

Mitigation:
- Perform testing during multiple time periods.
- Retest failures before logging defects.
- Capture API evidence using Postman and screenshots.

Risk 2: UI Changes in OrangeHRM

Description:
Page elements or locators may change and cause automation failures.

Mitigation:
- Use reliable locators where possible.
- Implement the Page Object Model (POM) design pattern.
- Maintain locator centralization for easier updates.

Risk 3: Environment or Browser Issues

Description:
Browser updates or environment restrictions may affect automation execution.

Mitigation:
- Use stable browser versions.
- Verify environment configuration before execution.
- Document environment details in test reports.
