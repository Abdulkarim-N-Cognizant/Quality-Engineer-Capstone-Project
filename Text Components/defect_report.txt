Defect ID: DEF-001

Title: POST /posts Accepts Request with Missing Required Fields
Environment: JSONPlaceholder API (https://jsonplaceholder.typicode.com/posts)
Severity: Medium
Priority: Medium
Status: New
Reported By: Abdulkarim Noorzaie
Date: July 7, 2026
Steps to Reproduce:

Send a POST request to https://jsonplaceholder.typicode.com/posts.
Use the following request body:
{
"title": "Test Post"
}
Submit the request.

Expected Result:
The API should return a 400 Bad Request response indicating that required fields are missing.
Actual Result:
The API returns a 201 Created response and accepts the request despite missing fields such as body and userId.
Root Cause (if known):
The API does not perform validation on required request fields before accepting the request.
Attachments:

Screenshot of the POST request in Postman
Screenshot of the response showing HTTP 201 Created
Request payload used during testing
Postman test execution results

Priority: Low
Status: New

Defect ID: DEF-003
Title: Password Field Does Not Mask Characters When Typing
Environment: OrangeHRM Demo Application
Severity: High
Priority: High
Status: New
Reported By: Maleesha Wanniarachchige
Date: July 7, 2026
Steps to Reproduce:

Navigate to the OrangeHRM login page.
Click inside the Password field.
Enter any password value.

Expected Result:
Characters entered into the Password field should be hidden and displayed as dots or asterisks.
Actual Result:
Characters entered into the Password field are displayed in plain text.
Root Cause (if known):
The password input field may be configured with the incorrect HTML input type.

Screenshot of the login page showing visible password characters
